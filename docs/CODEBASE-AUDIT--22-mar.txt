DATABASE SCHEMA (from migrations)
Tables (canonical state after all migrations applied)
clinic
- id UUID PK
- name text
- email text
- plan text (Starter | Pro | Enterprise)
- status text (active | inactive | trial)
- created_at timestamptz
actor (universal identity node — every person/entity is an actor)
- id UUID PK
- clinic_id UUID FK → clinic(id)
- name text
- type text (patient | user | clinic | system)
- created_at timestamptz
app_user (authentication identity, 1:1 with Supabase auth.users)
- id UUID PK = auth.uid()
- actor_id UUID FK → actor(id)
- email text
- role text (ADMIN | RECEPTION | DOCTOR | STOCK | MARKETING | IT | USER | SUPER_ADMIN | CONSULTANT | SUPPORT | ANALYST)
- iam_bindings JSONB (array of {roleId, resource, permissions?, denied?})
- assigned_room text (nullable)
patient
- id UUID PK (= actor.id via FK patient_id_fkey)
- clinic_id UUID FK → clinic(id)
- cpf text (nullable)
- birth_date date (nullable)
- gender text (nullable)
- Name comes from actor.name via join
appointment
- id UUID PK
- clinic_id UUID FK → clinic(id)
- patient_id UUID FK → patient(id) (nullable — legacy)
- patient_name text (denormalized, server-filled by RPC from actor)
- doctor_actor_id UUID FK → actor(id) (canonical, added in migration 20260321000000)
- appointment_date timestamptz
- status text (Agendado | Aguardando | Chamado | Em Atendimento | Realizado | Cancelado)
- room_number text (nullable)
- priority text (nullable)
- timestamp timestamptz
- Dropped legacy columns: doctor_id (text), doctor_name (text)
clinical_record
- id UUID PK
- clinic_id UUID FK → clinic(id)
- patient_id UUID FK → patient(id) (nullable)
- doctor_actor_id UUID FK → actor(id) (canonical)
- content text
- type enum record_type (EVOLUCAO | RECEITA | ATESTADO | EXAME — inferred)
- timestamp timestamptz
- Dropped legacy columns: doctor_id (text)
product (canonical inventory — absorbed inventory_item)
- id UUID PK
- clinic_id UUID FK → clinic(id)
- name text
- description text (nullable, added migration 20260321000001)
- unit text NOT NULL default 'un' (added migration 20260321000001)
- current_stock numeric
- min_stock numeric
- avg_cost_price numeric
- price numeric
- barcode text (nullable)
- category text (nullable)
- deleted boolean default false
stock_transaction (canonical movements — absorbed inventory_movement)
- id UUID PK
- clinic_id UUID FK → clinic(id)
- product_id UUID FK → product(id)
- type text (IN | OUT)
- total_qty numeric
- reason text (MANUAL | PURCHASE | PROCEDURE | ...)
- record_id UUID (nullable — links to procedure execution ID)
- actor_id UUID FK → actor(id) (nullable, added migration 20260321000001)
- notes text (nullable, added migration 20260321000001)
- timestamp timestamptz
- Trigger: trg_stock_transaction_sync — after INSERT, updates product.current_stock atomically
procedure_type
- id UUID PK
- clinic_id UUID FK → clinic(id)
- name text
- code text (nullable, TUSS or internal)
- price numeric
- active boolean
- created_at timestamptz
procedure_recipe
- id UUID PK
- procedure_type_id UUID FK → procedure_type(id) ON DELETE CASCADE
- item_id UUID FK → product(id) ON DELETE RESTRICT (repointed in migration 20260321000001)
- quantity numeric
- created_at timestamptz
social_post
- id UUID PK
- clinic_id UUID FK → clinic(id)
- title text
- content text
- platform text (instagram | facebook | linkedin)
- status text (draft | scheduled | published)
- image_url text (nullable)
- scheduled_at timestamptz (nullable)
- created_at timestamptz
access_request
- id UUID PK
- clinic_id UUID FK → clinic(id)
- clinic_name text
- reason text
- status text (pending | approved | denied)
- requested_role_id text
- requester_user_id UUID FK → app_user(id) ON DELETE CASCADE (canonical, added migration 20260321000000)
- created_at timestamptz
- Dropped legacy columns: requester_id (text), requester_name (text)
ui_module (migration 20260322000000)
- id UUID PK
- key VARCHAR(50) UNIQUE (inventory | reception | patients | appointments | procedures | clinical | reports | social)
- label VARCHAR(50)
- icon VARCHAR(50) (Lucide icon name string)

- description text
- created_at timestamptz

inventory_item, inventory_movement, "user" (legacy PascalCase user), "AccessRequest", "AccessBinding", "ApiKey", "Batch", "Contact", "FinancialCategory", "FinancialTransaction", "ProductHistory", "StockTransaction", batch, financial_transaction

CORE SERVICES — Every Method
SupabaseService
Thin wrapper around @supabase/supabase-js.
- get auth() — exposes supabase.auth
- from(table) — typed table accessor
- rpc(fn, args) — RPC call passthrough
AuthService
- login(email, password): Promise<boolean> — calls supabase.auth.signInWithPassword, returns success flag
- logout() — delegates to AuthStateService.logout()
AuthStateService
- Signal: currentUser: WritableSignal<UserProfile | null>
- initSession() — reads existing session on startup, subscribes to onAuthStateChange
- loadUserProfile(uid) — fetches app_user + joined actor, sets currentUser, auto-selects clinic context: SUPER_ADMIN → 'all', others → their clinicId
- checkPermission(permission, resource) — evaluates iam_bindings[] against IAM_ROLES permission sets; SUPER_ADMIN always returns true
- logout() — calls supabase.auth.signOut(), navigates to /login
ClinicContextService
- Signal: selectedClinic: WritableSignal<string | null> — the currently active clinic UUID or 'all'
- Computed: clinicId() — returns null when 'all', otherwise the UUID (used by services to gate queries)
PatientService
State signals: patients, appointments, clinicalRecords (all WritableSignal).  
Auto-syncs via effect() on context.clinicId() change.
- syncForClinic(clinicId) — parallel fetch of patient (joined actor), appointment (joined patient→actor), clinical_record (joined patient→actor); also fetches app_user for doctor name resolution
  - Reads: app_user, patient, appointment, clinical_record
- createPatient(patient) — calls RPC create_patient_with_actor; optimistic update to patients signal
  - Writes: actor, patient
- createAppointment(apt) — calls RPC add_appointment; optimistic update to appointments
  - Writes: actor (read), appointment
- updateAppointmentStatus(id, status) — direct appointment.update({status}), updates signal
  - Writes: appointment
- updateAppointmentRoom(id, roomNumber) — direct appointment.update({room_number}), updates signal
  - Writes: appointment
- transitionAppointmentStatus(id, newStatus) — validates state machine (Agendado→Aguardando→Chamado→Em Atendimento→Realizado, or any→Cancelado); calls updateAppointmentStatus
  - Writes: appointment
- createClinicalRecord(rec) — calls RPC add_clinical_record; optimistic prepend to clinicalRecords
  - Writes: clinical_record
PatientStore
Thin facade over PatientService. Exposes read-only computed signals + convenience mutation wrappers with loading/error signals.
Signals exposed:
- patients = computed(() => patientService.patients())
- records = computed(() => patientService.clinicalRecords())
- appointments = computed(() => patientService.appointments())
- loading: WritableSignal<boolean>
- error: WritableSignal<string | null>
- patientCount: computed
Methods:
- loadPatients(clinicId) — calls patientService.syncForClinic(clinicId)
- loadAppointments(clinicId) — calls patientService.syncForClinic(clinicId) (same call as above — intentional)
- loadClinicalRecords(patientId) — STUB: does nothing
- createPatient(patient) — wraps patientService.createPatient() with loading/error
- createAppointment(apt) — wraps patientService.createAppointment() with loading/error
- createRecord(rec) — wraps patientService.createClinicalRecord() with loading/error
- updateAppointmentStatus(id, status) — calls patientService.transitionAppointmentStatus() (uses state machine validator)
- updateAppointmentRoom(id, room) — calls patientService.updateAppointmentRoom()
InventoryService
Pure promise-based; no internal signals. All methods require a valid clinicId from ClinicContextService.
- getItems() — SELECT from product WHERE clinic_id AND deleted=false ORDER BY name. Reads: product
- getItem(id) — SELECT single from product. Reads: product
- createItem(item) — INSERT into product. Writes: product
- updateItem(id, updates) — UPDATE product. Writes: product
- deleteItem(id) — soft-delete: UPDATE product SET deleted=true. Writes: product
- recordMovement(movement) — calls RPC add_stock_movement; trigger updates product.current_stock. Writes: stock_transaction, product
- getMovements(productId?) — SELECT from stock_transaction JOIN product, actor. Reads: stock_transaction
- getProcedureTypes() — SELECT from procedure_type. Reads: procedure_type
- createProcedureType(proc) — INSERT into procedure_type. Writes: procedure_type
- updateProcedureType(id, updates) — UPDATE procedure_type. Writes: procedure_type
- getRecipes(procedureTypeId) — SELECT from procedure_recipe JOIN product. Reads: procedure_recipe, product
- addRecipeItem(recipe) — INSERT into procedure_recipe. Writes: procedure_recipe
- removeRecipeItem(id) — DELETE from procedure_recipe. Writes: procedure_recipe
- performProcedure(procedureTypeId, patientId, notes) — calls RPC perform_procedure; deducts all recipe ingredients. Writes: stock_transaction, product (trigger)
- getProcedureAuditLog() — SELECT from stock_transaction WHERE reason='PROCEDURE' LIMIT 20. Reads: stock_transaction
InventoryStore
Thin facade — exposes products and transactions signals from DatabaseService.
- products = db.products (WritableSignal, populated by DatabaseService.syncDataForClinic)
- transactions = db.transactions (WritableSignal)
- loading: WritableSignal<boolean> (stub, always false)
- loadProducts(clinicId) — STUB: does nothing (data auto-loads via DatabaseService effect)
- loadTransactions(clinicId) — STUB: does nothing
DatabaseService
Central facade aggregating all sub-services. Exposes most signals via computed/direct passthrough.
Signals:
- currentUser = computed(() => authState.currentUser())
- selectedContextClinic = context.selectedClinic (WritableSignal — directly exposed, writable from outside)
- patients, appointments, clinicalRecords — computed from patientService
- products, transactions — WritableSignal (populated in syncDataForClinic)
- users, clinics, socialPosts, accessRequests — WritableSignal
- globalArr, globalUptime — WritableSignal (hardcoded: 0, 99.9)
- accessibleClinics = computed(() => clinics())
Constructor effect: When selectedClinic changes and is not 'all', calls syncDataForClinic. If 'all', loads all clinics.
- syncDataForClinic(clinicId) — parallel fetch of: app_user (clinic staff), clinic (all), social_post, access_request, inventoryService.getItems(), inventoryService.getMovements()
  - Reads: app_user, clinic, social_post, access_request, product, stock_transaction
- checkPermission(permission, resource) — delegates to authState.checkPermission()
- logout() — delegates to authState.logout()
- addPatient(), addAppointment(), updateAppointmentStatus(), updateAppointmentRoom(), transitionAppointmentStatus(), addClinicalRecord() — all delegate to patientService
- addSocialPost(post) — INSERT into social_post, maps to SocialPost, updates socialPosts signal. Writes: social_post
- addProduct(product) — delegates to inventoryService.createItem(), updates products signal. Writes: product
- deleteProduct(id) — delegates to inventoryService.deleteItem(), updates products signal. Writes: product
- addTransaction(tx) — delegates to inventoryService.recordMovement(). Writes: stock_transaction, product
- requestAccess(clinicId, clinicName, reason, roleId) — INSERT into access_request. Writes: access_request
- approveAccess(reqId) — UPDATE access_request SET status='approved'. Writes: access_request
- addClinic(c) — INSERT into clinic. Writes: clinic
- deleteClinic(id) — UPDATE clinic SET status='inactive'. Writes: clinic
- assignRoom(room) — UPDATE app_user SET assigned_room, updates currentUser signal. Writes: app_user
- saveUser(u, pw?) — If existing: RPC update_user_with_actor, optionally supabase.auth.updateUser({password}); If new: supabase.auth.signUp() then RPC create_user_with_actor. Writes: actor, app_user, auth.users
UiConfigService
- State signal: configState: WritableSignal<ClinicUiConfig | null>
- Computed: allModules(), enabledModules(), config()
- Auto-fetches on context.clinicId() change
- fetchConfig(clinicId) — calls RPC get_clinic_ui_config. Reads: clinic_module, ui_module, clinic_config
- toggleModule(moduleKey, enabled) — UPDATE clinic_module, optimistic signal update. Writes: clinic_module
- updateConfigValue(key, value) — UPSERT clinic_config. Writes: clinic_config
GeminiService
Wraps @google/genai. API key from process.env['API_KEY'] (no fallback).
- connectLive(config) — opens Gemini Live session (real-time audio)
- generateContent(prompt, systemInstruction?) — gemini-2.0-flash, returns string
- analyzeInventoryRisks(products) — gemini-2.5-flash, Markdown bullet list
- generateStrategicInsights(context) — gemini-2.5-flash, Markdown insights
- generateSocialContent(topic, tone) — gemini-2.5-flash, JSON response {caption, hashtags, imageUrl}
- processClinicalAudio(audioData) — gemini-2.5-flash, multimodal audio + text, returns Markdown
LocalAiService
Wraps @mlc-ai/web-llm (WebLLM, runs in browser via WebGPU).
- Signals: status, progress, isLoaded, hasWebGPU
- availableModels — 3 models: Gemma-2b (low), Llama-3-8B (high), Phi-3-mini (medium)
- checkWebGPUSupport() — checks navigator.gpu, fallback via TensorFlow.js
- loadModel(modelId) — downloads and initializes MLCEngine
- unloadModel() — frees GPU memory
- generate(prompt) — chat completion with medical scribe system prompt
PrintService
- printElement(elementId, title?) — creates hidden iframe, injects Tailwind-compatible print CSS, copies element, triggers window.print(), removes iframe after 1s
CsvImportService
- parseCsv(file) — reads File/Blob, parses headers + rows (handles quoted comma-delimited values)
- mapToProduct(row, clinicId) — maps CSV columns (Produto, Valor de venda, Valor de custo, Estoque atualizado, Validade, Grupo, Código de barras, Observações) to Product
- detectConflicts(imported, existing) — matches by SKU then by name (normalized); returns array of {type: 'NEW'|'UPDATE', product, existing?, diffs?, matchType?}

Overlaps:
- InventoryStore.products (used by Reports) and InventoryComponent.items signal are separate signals fed from the same product table but NOT synchronized — a change in one does not update the other. InventoryComponent has its own local items = signal<InventoryItem[]>([]) and InventoryStore.products is fed separately by DatabaseService.syncDataForClinic. This is a data duplication / stale data bug.
What's missing/stubbed:
- Scanner lookup uses i.id === code — since id is a UUID, barcode scanners that scan EAN/Code128 barcodes will never match. The schema has a barcode field in product but InventoryService.getItems() returns the full row — the scanner should match against i.barcode, not i.id. The comment in the code acknowledges this but it remains broken
- formData.cost_price in the HTML template (line 234) binds to a non-existent field — formData has avg_cost_price, not cost_price. This means the cost price field in the edit form never saves
- Import button shown in UI (Upload icon referenced in icons list) but no import functionality in this component — CsvImportService and ImportConflictModalComponent exist but are not used here
- No real-time updates
- viewFilter = 'critical' works but expiryDate is not in InventoryItem type — the Reports component checks p.expiryDate via InventoryStore.products (which uses the Product type that has expiryDate) but InventoryComponent uses InventoryItem type which does not have expiryDate — different type mappings for the same data
- current_stock is set directly in form and passed to createItem — but the DB trigger will handle stock updates on movements; setting it at creation is the only time it's directly written (correct for initial stock)
